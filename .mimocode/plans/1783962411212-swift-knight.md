### 2. Architecture Overview
- Laravel 12 backend (PHP 8.2+) + Vue 3 SPA frontend
- Single catch-all web route renders `app.blade.php` shell; all data via `/api/*`
- File-based Vue routing via `vite-plugin-pages` (pages in `resources/js/pages/`)
- Vite alias: `@` = `resources/js`

### 3. Key Gotchas an Agent Would Miss
- **No JS linter/formatter** — no ESLint, Prettier, or TS config exists. Only PHP has Pint.
- **No type checking** — no PHPStan, Psalm, or TypeScript. PHP and JS are both untyped.
- **Tests are placeholders** — only `ExampleTest.php` stubs exist. No real test coverage.
- **RefreshDatabase is commented out** in `tests/Pest.php` — tests don't auto-reset DB state.
- **Queued mailables** — `DonationReceipt` implements `ShouldQueue`. Queue worker must be running to send emails.
- **Dynamic config from DB** — `SettingsServiceProvider` overrides `config()` at boot. Stripe/SSLCommerz/mail settings come from the `settings` table, not `.env`.
- **Dual payment gateways** — SSLCommerz (BDT) for Bangladesh, Stripe (USD) for international. Routing logic is in `DonationController`.
- **Bilingual content** — UI strings via vue-i18n (`en.json`/`bn.json`). Content fields stored as duplicate columns (`title`/`title_bn`). Use `useLangStore().f()` to pick the right field.
- **Legacy `fbf/` directory** — static HTML from an earlier version, not part of the Vue app.

### 4. Project Structure (Key Paths Only)
- `app/Http/Controllers/Admin/` — 11 admin controllers
- `app/Http/Controllers/` — 8 public controllers
- `app/Services/` — `StripeService`, `SSLCommerzService`
- `app/Models/` — Eloquent models (User, Project, Donation, etc.)
- `resources/js/pages/` — file-based Vue routes (source of truth for routing)
- `resources/js/stores/` — 4 Pinia stores (auth, admin, lang, theme)
- `routes/api.php` — all REST endpoints
- `database/migrations/` — 20 migration files
- `database/seeders/` — RolePermissionSeeder, ContentSeeder, SettingsSeeder

### 5. Docker
- `docker-compose.yml` with 4 services: app (PHP-FPM), web (Nginx), db (MySQL 8), node (Node 22)
- Custom configs in `docker/nginx.conf` and `docker/php.ini`

### 6. RBAC
- Custom implementation (not Spatie). Models: User, Role, Permission with pivots.
- 4 roles (admin, manager, editor, user) + 27 permissions seeded by `RolePermissionSeeder`.
- Admin access checked by `AdminMiddleware`. Fine-grained by `PermissionMiddleware`.
